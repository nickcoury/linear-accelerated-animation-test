How to use: Jank is on by default — the main thread is spin-locked for ~500 ms every 500 ms. If your browser compositor-accelerates linear(0, 1), both boxes stay smooth. If it doesn't, the yellow box stutters while the green one keeps gliding. Click Disable Jank to compare with a clean main thread.

What's happening: linear(0, 1) produces a straight-line easing identical to the linear keyword. The question is whether the browser's animation optimizer recognizes this equivalence and promotes the animation to the compositor thread (which runs independently of JS on the main thread), or whether it falls back to main-thread animation because it doesn't parse the linear() function as a trivially fast path.
